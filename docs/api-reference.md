# API Reference

Freki is the public HTTP API for Grimnir. It also serves Hlidskjalf, the
single-file dashboard, when the frontend directory is present in the container.

At runtime, FastAPI also exposes generated OpenAPI and browser docs:

| Path | Purpose |
|------|---------|
| `/openapi.json` | Generated OpenAPI schema from FastAPI. |
| `/docs` | Swagger UI generated from the current app. |
| `/redoc` | ReDoc generated from the current app. |

These generated pages are useful for exact request and response schemas. This
document records the stable product surface and integration behavior.

## Service Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/health` | None | Freki liveness check. Returns `{"status": "ok"}`. |
| GET | `/metrics` | None | Prometheus metrics generated by `prometheus-fastapi-instrumentator`. Excluded from the OpenAPI schema. |

## Receiver Status and CSI Streams

| Method | Path | Query or body | Description |
|--------|------|---------------|-------------|
| GET | `/api/stream` | None | Server-Sent Events snapshot of active receivers and recent RSSI stats once per second. |
| GET | `/api/csi-stream` | None | Server-Sent Events tail of raw CSI rows. Starts at the current tail and emits new rows. Used by Volva. |
| GET | `/api/history/receivers` | None | Lists receivers with heartbeat data. |
| GET | `/api/history/variance` | `receiver_id` required, `minutes` optional, 1 to 1440, default 60 | Returns per-minute RSSI variance. Falls back to raw samples if the continuous aggregate is still sparse. |
| GET | `/api/history/snapshot` | `receiver_id` required, `limit` optional, 1 to 50, default 1 | Returns recent raw CSI samples for heatmap rendering. |

SSE endpoints use `text/event-stream`, disable proxy buffering with
`X-Accel-Buffering: no`, and send keepalive comments when idle where implemented.

## Rooms and Labels

| Method | Path | Query or body | Description |
|--------|------|---------------|-------------|
| GET | `/api/rooms` | None | Lists rooms ordered by floor and name. |
| POST | `/api/rooms` | JSON `{"name": string, "floor": integer}` | Creates a room. Returns `409` when the room already exists. |
| PATCH | `/api/rooms/{room_name}` | JSON with optional `name` and `floor` | Updates room metadata. Name changes cascade to labels and update `csi_samples.label`. |
| DELETE | `/api/rooms/{room_name}` | None | Deletes a room. Returns `409` when labels still reference it. |
| GET | `/api/labels` | `minutes` optional, default 120 | Lists labels whose end time is within the requested window. |
| POST | `/api/labels` | JSON `time_start`, `time_end`, `room`, `occupants`, optional `notes` | Creates a label and schedules best-effort backfill into CSI samples and training samples. |
| DELETE | `/api/labels/{label_id}` | None | Deletes a label, clears its backfill window, then reapplies overlapping labels. |

Label creation requires `time_end` to be after `time_start`; unknown rooms
return `422`.

## Training Data, Daemons, and Jobs

Nornir uses these endpoints to register itself, claim jobs, fetch data, and
report status.

When `ML_CONTROL_SHARED_SECRET` is configured, machine-control writes require
the `X-Grimnir-ML-Control-Secret` header. This applies to daemon heartbeat and
training job claim, heartbeat, complete, and fail calls.

| Method | Path | Query or body | Description |
|--------|------|---------------|-------------|
| GET | `/api/training-data` | `time_start`, `time_end`, `rooms`, optional `cursor`, optional `page_size` 1 to 5000 | Cursor-paginated labeled CSI export. `rooms` is a comma-separated list with 1 to 32 entries. |
| GET | `/api/training-daemons` | None | Lists registered training daemons. |
| POST | `/api/training-daemons/heartbeat` | JSON `name`, `host`, optional `ip_address`, optional `capabilities` | Upserts a daemon row and updates `last_seen`. Requires ML control secret when configured. |
| GET | `/api/training-jobs` | Optional `status`, optional `limit`, default 50, max 500 | Lists training jobs. |
| POST | `/api/training-jobs` | JSON `{"spec": {...}}` | Enqueues a job. Validates time order and room names. |
| POST | `/api/training-jobs/{job_id}/claim` | JSON `{"daemon_id": integer}` | Claims a queued job with a race-free conditional update. Requires ML control secret when configured. |
| POST | `/api/training-jobs/{job_id}/heartbeat` | JSON `daemon_id`, `claim_token` | Updates heartbeat for the owning daemon and claim token. |
| POST | `/api/training-jobs/{job_id}/complete` | JSON `daemon_id`, `claim_token` | Marks the owned running job complete and clears the claim token. |
| POST | `/api/training-jobs/{job_id}/fail` | JSON `daemon_id`, `claim_token`, `error` | Marks the owned running job failed and clears the claim token. |
| POST | `/api/training-jobs/{job_id}/cancel` | None | Cancels a queued or running job. |

Training jobs use these statuses: `queued`, `running`, `failed`, `complete`,
and `cancelled`.

## Models

When `MODEL_UPLOAD_SHARED_SECRET` is configured, `POST /api/models` requires
the `X-Grimnir-Model-Upload-Secret` header. Nornir sends it automatically when
the same environment variable is configured for the daemon.

| Method | Path | Query or body | Description |
|--------|------|---------------|-------------|
| GET | `/api/models` | Optional `limit`, default 50, max 500 | Lists model metadata without loading model bytes. |
| GET | `/api/models/active` | None | Returns active model metadata or `404` when no model is active. |
| GET | `/api/models/{model_id}/data` | None | Downloads the raw serialized model bytes as `application/octet-stream`. |
| POST | `/api/models` | `multipart/form-data`: `name`, `metrics` JSON string, `feature_config` JSON string, optional `training_job_id`, `model_data` file | Uploads a trained model. Validates JSON and feature config. |
| POST | `/api/models/{model_id}/activate` | None | Atomically marks one model active and clears the previous active model. |

## Predictions

Volva writes current prediction state to Freki. Home Assistant and dashboards
can read or subscribe to the Freki state.

| Method | Path | Query or body | Description |
|--------|------|---------------|-------------|
| PUT | `/api/predictions/current` | JSON `timestamp`, `model_id`, `rooms` | Stores the latest prediction envelope in Postgres. Returns `204`. |
| GET | `/api/predictions/current` | None | Returns the current prediction envelope or `404` if none has been published. |
| GET | `/api/predictions/stream` | None | SSE stream that emits whenever the current prediction row changes. |

Prediction envelope shape:

```json
{
  "timestamp": "2026-04-17T12:00:00Z",
  "model_id": 3,
  "rooms": {
    "kitchen": {
      "human_count": 1
    }
  }
}
```

## Volva Service Endpoints

Volva has no public prediction API. It dials Freki, consumes `/api/csi-stream`,
and writes `/api/predictions/current`.

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Returns service status, active model id, and model age. |
| GET | `/metrics` | Prometheus metrics for active model state and HTTP instrumentation. |
